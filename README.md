# Eurobot-2026

ROS 2 система управления для робота на соревнованиях Eurobot 2026.

## Структура репозитория

| Директория | Описание |
|------------|----------|
| esp32_low_level/ | Прошивка для ESP32 (моторы, сервоприводы, лидар, одометрия) |
| esp32_bridge/ | ROS 2 мост для связи с ESP32 через TCP сокеты |
| my_nav_launch/ | Launch-пакет для запуска навигации Nav2 |
| my_teleop/ | Клавиатурный телеоп для управления роботом |
| route_follower/ | Модуль следования по траектории |
| configs/ | Конфигурационные файлы |
| route_follower/ | Утилита для последовательного выполнения маршрутов, заданных в JSON-файлах (подъёмник, паузы). |

---

## 🚀 Общий порядок запуска

Для полноценной работы системы необходимо запустить узлы в следующем порядке.

### 1. Подготовка окружения

Убедитесь, что все ROS 2 пакеты собраны.

```bash
colcon build --packages-select esp32_bridge my_teleop my_nav_launch route_follower
source install/setup.bash
```

### 2. Запуск прошивки на ESP32

Загрузите прошивку из папки `esp32_low_level` на плату. Убедитесь, что ESP32 подключена к той же WiFi-сети, что и ваш компьютер с ROS 2.

### 3. Запуск моста `esp32_bridge`

```bash
ros2 run esp32_bridge esp32_bridge.py --ros-args -p host:=<IP_адрес_ESP32>
```

### 4. Запуск системы навигации

```bash
ros2 launch my_nav_launch navigation.launch.py \
    map_file:=/путь/к/карте/map.yaml \
    config_file:=/путь/к/конфигу/nav_param.yaml
```

### 5. (Опционально) Ручное управление

```bash
ros2 run my_teleop my_teleop.py
```

### 6. (Опционально) Выполнение маршрута

Для автоматического выполнения серии задач (например, перемещение подъёмника, паузы) запустите `route_follower`.

```bash
ros2 run route_follower route_follower --ros-args -p route_file:=/путь/к/маршруту.json
```

## Возможности системы

### ESP32 Low Level
- Управление шаговыми двигателями колёс
- Управление подъёмником с синхронизацией двух моторов
- Управление 4 сервоприводами с плавным перемещением
- Обработка данных с лидара (LDS) с проверкой CRC
- Расчёт одометрии на основе шаговых двигателей
- TCP сервер для связи с ROS2
- Автоматическое переподключение клиентов
- Отправка состояния кнопки старта и переключателя стороны

### ROS 2 Bridge
- Публикация одометрии (nav_msgs/Odometry)
- Публикация сканов лидара (sensor_msgs/LaserScan)
- Публикация состояния подъёмника и сервоприводов
- Подписка на команды скоростей (geometry_msgs/Twist)
- Статические и динамические TF трансформации
- Автоматическое переподключение при обрыве связи
- Поддержка лидарного протокола с CRC16

### Teleop
- Управление движением (WASD клавиши)
- Управление сервоприводами (цифровые клавиши)
- Управление подъёмником (Q/E, R/F)
- Остановка робота (пробел)

### Nav2 Интеграция
- Автоматическое планирование маршрутов
- Глобальная и локальная costmap
- Regulated Pure Pursuit контроллер
- Поддержка карт формата PGM/YAML
- Lifecycle управление узлами

### Топики ros2

| Топик | Тип | Направление | Описание |
| :--- | :--- | :--- | :--- |
| `/pwb/cmd_vel` | `geometry_msgs/Twist` | Подписка | Команды линейной и угловой скорости |
| `/pwb/odom` | `nav_msgs/Odometry` | Публикация | Данные одометрии робота |
| `/pwb/lidar_scan` | `sensor_msgs/LaserScan` | Публикация | Данные с лидара |
| `/pwb/lift_target_height` | `std_msgs/UInt16` | Публикация / Подписка | Целевая высота подъёмника |
| `/pwb/lift_current_height` | `std_msgs/UInt16` | Публикация | Текущая высота подъёмника |
| `/pwb/servos_target_angles` | `std_msgs/UInt8MultiArray` | Публикация / Подписка | Целевые углы сервоприводов (0–90°) |
| `/pwb/servos_current_angles` | `std_msgs/UInt8MultiArray` | Публикация | Текущие углы сервоприводов |
| `/initialpose` | `geometry_msgs/PoseWithCovarianceStamped` | Подписка | Сброс одометрии в начальную позицию |

### Фреймы TF

| Фрейм | Описание |
|-------|----------|
| map | Глобальная система координат |
| pwb_odom | Одометрическая система координат |
| base_link | Базовая система координат робота |
| lidar | Система координат лидара |
| lift | Система координат подъёмника |
| servo0, servo1, servo2, servo3 | Системы координат сервоприводов |

## Коммуникационный протокол

Порт 8080 — управление роботом
Порт 8090 — данные лидара
Формат пакетов: [2 байта: длина][1 байт: тип][данные]

## ⚙️ Конфигурация и параметры

### Основные параметры `esp32_bridge`

| Параметр | Значение по умолчанию | Описание |
| :--- | :--- | :--- |
| `host` | `127.0.0.1` | IP-адрес ESP32 |
| `port` | `8080` | Порт для команд управления |
| `lidar_port` | `8090` | Порт для данных лидара |
| `odom_frame` | `pwb_odom` | Имя фрейма одометрии |
| `base_frame` | `base_link` | Имя базового фрейма робота |
| `lidar_frame` | `lidar` | Имя фрейма лидара |
