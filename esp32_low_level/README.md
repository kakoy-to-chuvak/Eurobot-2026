# ESP32 Low Level Firmware
Прошивка для ESP32 робота. Обеспечивает управление моторами, подъёмником, сервоприводами и лидаром через TCP сокеты.

## Структура файлов

| Файл | Описание |
|------|----------|
| `esp32_low_level.ino` | Главный файл, инициализация и основной цикл |
| `parameters.h` | Конфигурационные параметры (пины, скорости, пределы) |
| `pinout.h` | Определения пинов ESP32 |
| `motors.ino` | Управление двигателями колёс |
| `odometry.ino` | Расчёт одометрии |
| `lift.ino` | Управление подъёмником (лифтом) |
| `servo_control.ino` | Управление сервоприводами |
| `lidar.ino` | Чтение и отправка данных лидара |
| `server.ino` | TCP сервер для связи с ROS мостом |

## Параметры (parameters.h)

### Настройки сети

| Параметр | Значение | Описание |
|----------|----------|----------|
| `SSID` | robotx | Имя WiFi сети |
| `PASSWORD` | 78914040 | Пароль WiFi |
| `CREATE_ACCESS` | 0 | Режим AP (1 - точка доступа, 0 - клиент) |
| `SERVER_PORT` | 8080 | Порт TCP сервера |
| `LIDAR_PORT` | 8090 | Порт лидарного сервера |

### Механические параметры

| Параметр | Значение | Описание |
|----------|----------|----------|
| `WHEEL_R` | 0.0455 м | Радиус колеса |
| `WHEEL_BASE` | 0.2066 м | Колёсная база |
| `WHEEL_STEPS_PER_REV` | 800 | Шагов на оборот мотора |
| `LIFT_MIN_POSITION` | 0 мм | Минимальная высота подъёмника |
| `LIFT_MAX_POSITION` | 190 мм | Максимальная высота подъёмника |
| `LIFT_STEPS_PER_REV` | 800 | Шагов на оборот мотора подъёмника |

### Таймеры

| Параметр | Значение (мс) | Описание |
|----------|---------------|----------|
| `SERVER_DELAY` | 5 | Частота обработки TCP сервера |
| `ODOMETRY_DELAY` | 50 | Частота расчёта одометрии |
| `SENSORS_DELAY` | 50 | Частота опроса датчиков |

## Коммуникационный протокол

### Формат сообщения

```
[2 байта: длина пакета][1 байт: тип события][данные]
```

### Типы событий

| Направление | Событие | Код | Данные |
|-------------|---------|-----|--------|
| ROS → ESP | GET_ALL | 10 | - |
| ROS → ESP | GET_MOTORS_SPEED | 11 | - |
| ROS → ESP | GET_LIFT_HEIGHT | 12 | - |
| ROS → ESP | GET_SERVO_STATE | 13 | - |
| ROS → ESP | GET_ODOMETRY | 14 | - |
| ROS → ESP | SET_MOTORS_SPEED | 71 | 2×float (linear, angular) |
| ROS → ESP | SET_LIFT_HEIGHT | 72 | uint16_t (height_mm) |
| ROS → ESP | SET_SERVO_STATE | 73 | 4×uint8_t (angles) |
| ROS → ESP | SET_ODOMETRY | 74 | 3×float (x, y, theta) |
| ESP → ROS | ANSWER_GET_ALL | 40 | см. структуру ниже |
| ESP → ROS | ANSWER_GET_MOTORS_SPEED | 41 | 2×float |
| ESP → ROS | ANSWER_GET_LIFT_HEIGHT | 42 | uint16_t |
| ESP → ROS | ANSWER_GET_SERVO_STATE | 43 | 4×uint8_t |
| ESP → ROS | ANSWER_GET_ODOMETRY | 44 | 3×float (theta, x, y) |
| ESP → ROS | SEND_START | 112 | uint8_t (0/1) |
| ESP → ROS | SEND_SIDE | 111 | uint8_t (0-yellow/1-blue) |

### Структура ANSWER_GET_ALL

| Порядок | Тип | Поле |
|---------|-----|------|
| 1 | float | linear_speed |
| 2 | float | angular_speed |
| 3 | uint16_t | lift_height |
| 4-7 | 4×uint8_t | servo_state |
| 8 | float | theta |
| 9 | float | x |
| 10 | float | y |

## Лидарный протокол

### Формат пакета лидара

```
[2 байта: длина пакета][N×20 байт: данные][2 байта: CRC16]
```

### Формат блока данных (20 байт)

| Смещение | Тип | Описание |
|----------|-----|----------|
| 0-1 | uint16_t | Начальный угол × 100 |
| 2-3 | uint16_t | Конечный угол × 100 |
| 4-5 | uint16_t | Дистанция 1 × 1000 (мм) |
| 6-7 | uint16_t | Дистанция 2 × 1000 |
| 8-9 | uint16_t | Дистанция 3 × 1000 |
| 10-11 | uint16_t | Дистанция 4 × 1000 |
| 12-13 | uint16_t | Дистанция 5 × 1000 |
| 14-15 | uint16_t | Дистанция 6 × 1000 |
| 16-17 | uint16_t | Дистанция 7 × 1000 |
| 18-19 | uint16_t | Дистанция 8 × 1000 |

## Режимы работы

### Режим клиента WiFi (по умолчанию)
Подключается к существующей WiFi сети.

```cpp
#define CREATE_ACCESS 0
#define SSID "имя_сети"
#define PASSWORD "пароль"
```

### Режим точки доступа
Создаёт собственную WiFi сеть.

```cpp
#define CREATE_ACCESS 1
#define MY_SSID "robotx_ap"
#define MY_PASSWORD "12345678"
```

## Отладка

### Уровни логирования

В `parameters.h` установите `LOG_LEVEL`:

| Уровень | Вывод |
|---------|-------|
| 2  | Только ошибики (ERROR) |
| 1  | + Предупреждения (WARN) |
| 0  | + INFO (базовая информация) |
| -1 | + DEBUG (отладка) |
| -2 | + TRACE (трассировка всех функций) |
| -3 | + TRACE2 (трассировка почти каждого блока кода) |
