# route_follower

ROS2 нода для последовательного выполнения маршрутов из JSON файла. Поддерживает различные типы задач (лифт, ожидание и др.).



## Запуск

### Базовый запуск

`
ros2 run route_follower route_follower --ros-args -p route_file:=/path/to/route.json
`

## Параметры

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| route_file | string | "" | Путь к JSON файлу с маршрутом |

## Формат JSON файла

Файл маршрута представляет собой массив задач. Каждая задача — объект с одним ключом (тип задачи) и значением (параметры).

### Общая структура

```json
[
    {"тип_задачи": {"параметр1": значение1, "параметр2": значение2, "wait": true}},
    {"тип_задачи2": {"параметр": значение}}
]
```

### Параметр wait

Для каждой задачи можно указать флаг `wait` (по умолчанию true):
- `true` — нода ожидает завершения задачи перед переходом к следующей
- `false` — задача запускается параллельно с последующими

## Поддерживаемые типы задач

### move_lift — управление подъёмником

Перемещает подъёмник на указанную высоту.

```json
{"move_lift": {"height": 100, "wait": true}}
```

| Параметр | Тип | Описание |
|----------|-----|----------|
| height | int | Целевая высота подъёмника (мм). Диапазон: 0-190 |

**Публикуемый топик:** `/pwb/lift_target_height` (std_msgs/UInt16)

**Подписываемый топик:** `/pwb/lift_current_height` (std_msgs/UInt16)

### sleep — пауза

Приостанавливает выполнение на указанное время.
```json
{"sleep": {"sleep": 2.5, "wait": true}}
```

| Параметр | Тип | Описание |
|----------|-----|----------|
| sleep | float | Длительность паузы (секунды) |

## Остановка

Для корректного завершения нажмите Ctrl+C. Нода автоматически завершит работу после выполнения всех задач.
